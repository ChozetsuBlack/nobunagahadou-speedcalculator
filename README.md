# 戦法スピード計算機 再現版

## 概要
このフォルダには、公開されていた Calconic ベースの計算サイトを解析し、独立動作する形で再現したサイト一式が入っています。

元サイトで確認できた入力定義と計算式をもとに、以下の3ファイル構成で再実装しています。
- `index.html`
- `style.css`
- `script.js`

## 現在のUIコンセプト
今回の画面デザインは、以下の方向性で再構成しています。
- ダークゲーム風
- 近未来HUD風
- 高級感のある攻略ツールUI
- 結果が主役のレイアウト
- スマホでも崩れにくい構成

配色は濃紺〜黒ベースで、アクセントに水色と紫を限定的に使用しています。

## このサイトでできること
- 戦法発動時間の即時計算
- 速度補正・ゲージ加算補正の可視化
- 計算根拠の表示
- 入力プリセットの呼び出し
- テストケース一覧からの入力反映
- URLパラメータによる状態保存と復元
- 共有URLのコピー

## 計算対象
出力は `発動時間` です。
表示は小数第2位固定です。

計算は以下の考え方で行っています。
1. ゲージ加算系で基本戦法間隔を短縮する
2. 状態変化の持続時間区間だけ、状態変化込み速度倍率を適用する
3. 残り時間に常時速度倍率を適用して最終発動時間を求める

## 使用方法
### 1. 開き方
`index.html` をブラウザで開いてください。
同じフォルダにある `style.css` と `script.js` を自動で読み込みます。

### 2. 基本操作
- スライダーを動かす
- ラジオボタンを選ぶ
- チェックボックスをオン/オフする

操作すると、結果は自動で再計算されます。

### 3. プリセットの使い方
プリセットは、現在はヒーローではなく右側の `Quick Access` エリアに配置しています。

現在用意しているプリセット:
- 初期状態
- 25秒最速寄せ
- 30秒バランス
- 35秒デバフ検証

ボタンを押すと、対応する入力状態がまとめて反映されます。

### 4. 計算根拠の見方
右側の `Analysis` パネルに以下を表示しています。
- 合計ゲージ加算
- 常時速度補正
- 状態変化補正
- ゲージ短縮後の基準時間
- 状態変化が効く区間
- 常時速度倍率
- 状態変化込み倍率
- 計算式の3段階表示

これにより、最終結果がどのように出ているかを追えるようにしています。

### 5. テストケース一覧の使い方
画面下部にテストケース一覧があります。
各ケースには以下を表示しています。
- ケースID
- 概要
- 計算結果
- `反映` ボタン

`反映` を押すと、そのケースの入力値が画面にセットされます。

### 6. URL共有機能
入力値を変更すると、現在の状態がURLパラメータに自動保存されます。

使い方:
- 入力を調整する
- `URLをコピー` を押す
- そのURLを別の人に渡す
- URLを開くと同じ入力状態が復元される

補足:
- URLにはゼロ以外の値だけを保存しています
- 初期状態に近いほどURLは短くなります

## 計算式
### 抽出できた元式
公開HTML内の `formula` 要素から、以下の式を確認しています。

```text
((#20*(100-(#6+#26+#28+#27+#25+#40+#41))/100)-(#34*(100+#13+#15+#16+#17+#18+#19+#29+#30+#35+#36+#37+#38+#39+#31+#33)/100))/((100+#13+#15+#16+#17+#18+#19+#29+#30+#35+#36+#37+#38+#39)/100)+#34
```

### 再現コード内での分解式
```text
gaugeTotal = baseGauge + boiTaisho + boiFukusho + kongo + bouga + building + shinki
baseSpeedTotal = kiryaku + shinsan + kanpa + jinsoku + sairyaku + reiken + ichinen + buretsu + fubu + kanata + sokkou + kenbatsu + sensokukashin
statusSpeedTotal = statusUp + statusDown

remainingGaugeTime = interval * (100 - gaugeTotal) / 100
baseMultiplier = (100 + baseSpeedTotal) / 100
statusMultiplier = (100 + baseSpeedTotal + statusSpeedTotal) / 100
buffedSegment = statusDuration * statusMultiplier
result = (remainingGaugeTime - buffedSegment) / baseMultiplier + statusDuration
```

### 解釈
- `gaugeTotal` は基本戦法間隔を短縮する要素です
- `baseSpeedTotal` は常時有効な速度補正です
- `statusSpeedTotal` は持続時間付きの状態変化です
- `statusDuration` の区間だけ `statusMultiplier` が使われます
